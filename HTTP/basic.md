## 웹 클라이언트와 서버

- 웹은 HTTP 라는 프로토콜(약속)을 사용한 통신으로 이루어져 있다.
- 웹 서버는 인터넷의 데이터를 저장하고 HTTP 클라이언트가 요청한 데이터를 제공함
- 클라이언트가 서버에게 HTTP요청을 보내면 서버는 요청된 데이터를 HTTP응답으로 돌려줌

<br>

## TCP/IP

- **프로토콜** : 컴퓨터와 네트워크 기기가 상호간에 통신하기 위한 규칙
- **TCP/IP란?** : 인터넷에 관련된 다양한 **프로토콜 집합의 총칭**. HTTP는 그중 하나

### TCP/IP는 4계층으로 이루어짐

- 계층화된 이유?
  - 사양이 변경되면 해당 계층만 바꾸면됨
  - 설계가 편함 (자기 자신이 담당하는 부분만 고려)

#### 애플리케이션 계층

- 유저에게 제공되는 애플리케이션에서 사용하는 **통신의 움직임 결정**
- **FTP, DNS**도 애플리케이션의 한가지. **HTTP**도 이 계층에 포함됨

#### 트랜스포트 계층

- 애플리케이션 계층에 네트워크로 접속된 2대의 컴퓨터 사이의 **데이터 흐름을 제공**
- **TCP와 UDP** 프로토콜이 있음

#### 네트워크 계층

- 네트워크 상에서 **패킷의 이동을 다룬다**
- 어떤 경로를 거쳐 상대 컴퓨터까지 패킷을 보낼지 결정

> 패킷 : 전송하는 데이터의 최소단위

#### 링크 계층

- 네트워크에 접속하는 **하드웨어적 면을 다룸**
- 디바이스 드라이버랑 _네트워크 인터페이스 카드(NIC)_ 를 포함
- 케이블 등과 같은 물리적으로 보이는부분도 포함

> 네트워크 인터페이스 컨트롤러(NIC) : 컴퓨터를 네트워크에 연결하여 통신하기 위해 사용하는 하드웨어 장치

<br>

## TCP/IP 통신의 흐름

#### 송신 : 계층을 통과할때마다 헤더 추가

1. 클라이언트의 **애플리케이션 계층**에서 어느 웹페이지를 보고싶다는 **HTTP** 리퀘스트 지시
2. **트랜스포트 계층(TCP)** 에서는 애플리케이션 계층에서 받은 데이터(HTTP 메세지)를 통신하기 쉽게 조각내서, 안내번호와 포트번호를 붙여 네트워크 계층에 전달
3. **네트워크 계층(IP)** 에서는 수신지 MAC 주소를 추가해서 **링크 계층**에 전달

#### 수신 : 계층을 통과할때마다 헤더 삭제

링크 계층에서 데이터를 받아들여 순서대로 위의 계층에 전달

> 헤더 : 해당 계층에 필요한 정보

<br>

## IP/TCP/DNS 프로토콜

### IP(Internet Protocol) : 네트워크 층에 해당

- 역할: 개개의 패킷을 상대방에게 전달
- 전달에 필요한 요소는 **IP 주소, MAC 주소**
  - IP 주소는 각 노드에 부여된 주소
  - MAC 주소는 각 네트워크 카드에 할당된 고유 주소
- IP주소는 변경 가능, MAC 주소는 변경불가

> 노드 : 네트워크에 연결된 컴퓨터와 장비들

```
IP주소가 기숙사라면 MAC주소는 학번이다.
IP 주소는 보낸 주소부터 도착지까지 경로를 찾기위한 주소고, MAC 주소는 한 주소내에 다양한 기기들 있을때, 각각 어떤 기기인지 식별하기위한 주소
```

- IP 통신은 **MAC 주소에 의존하여 통신**
  - 이때 ARP 프로토콜(주소 결정 프로토콜) 사용
- 컴퓨터와 라우터 등의 네트워크 기기는 목적지 도착전까지 대략적 목적지만 알고있다
  - 택배 배송과 비슷: `보낸사람->택배집배소->어느집` (전체과정을 모른다)

<br>

### TCP(Transfer Control Protocol) : 트랜스포트 층에 해당

- 대용량의 데이터를 보내기 쉽게 작게 분해해서 보내고, 정확히 도착했는지 확인하는 역할
- 상대에게 확실해 보내기 위해 TCP는 **쓰리웨이 핸드셰이킹** 사용

<br>

### DNS(Domain Name System)

- **도메인 명에서 IP 주소를 조사**하거나, **IP 주소로부터 도메인 명을 조사함**

<br>
 
## URL

- 통합 자원 지시자(Uniform Resource Locator, URL)는 리소스 식별자의 가장 흔한 형태
- URL은 특정 서버의 한 **리소스에 대한 구체적 위치**를 서술
- 리소스가 정확히 어디에 있고 어떻게 접근할 수 있는지 알려줌
- URL은 **세부분**으로 이뤄진 표준 포맷을 따름
  - 첫번째 부분은 **스킴(scheme)** 이라고 불림. 리소스에 접근하기 위해 사용되는 **프로토콜**을 서술. 보통 HTTP 프로토콜(http://)
  - 두번째 부분은 서버의 **인터넷 주소**를 제공(ex: www.ingg.com)
  - 세번째 부분은 웹서버의 **리소스** (ex: /datastructure/stack.gif)
- 오늘날의 대부분 URI는 URL임
