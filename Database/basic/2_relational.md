## 관계형 데이터베이스란

#### 현재 주류를 이루는 데이터베이스

#### 관계형 데이터베이스(Relational Database, RDB)

- 여기서의 `관계`란 2차원 표를 표기할 때 사용하는 단어
  - 가장 익숙한 2차원 표는 Excel이나 Google Docs의 스프레드시트
- 단적으로 말해서, 데이터를 2차원 표를 이용해 관리하는 데이터베이스

#### 관계형 데이터베이스의 이점

- 사람들이 자연스럽고 직관적으로 이해하기 쉬운 형식으로 데이터를 관리할 수 있다는 것
- 프로그래밍 언어를 사용하지 않아도 데이터를 조작할 수 있음
  - 이것을 가능하게한 것이 관계형 데이터베이스에 있는 `SQL`이라는 언어

<br>

## SQL 기초 지식

#### SQL 이란

- SQL(Structured Query Language) : 관계형 데이터베이스가 데이터를 조작하기 위해 준비한 언어

#### 4가지 기본 조작

- SELECT (검색)
- INSERT (등록)
- UPDATE (갱신)
- DELETE (제거)

#### 테이블, 행, 열

- 테이블 : 2차원 표. 관계형 데이터베이스에서 데이터를 관리하기 위한 유일한 단위
  - 어떤 테이블에 어떤 데이터를 포함하는가는 시스템 기능을 좌우하는 중요한 의미
- 열, 행 : 열(column, 컬럼), 행(row)

## 관계형 데이터베이스 사전 지식

#### 관계형 데이터베이스 소프트웨어

- 실제로 관계형 데이터베이스를 이용하려면 MySQL, Oracle 같은 특정 소프트 웨어 제품이 필요
- 무료로 사용 가능한 MySQL 이나 PostgreSQL 같은 오픈소스 제품이 인기

#### DBMS와 데이터베이스 차이

- `데이터베이스`는 기능이나 구조를 나타내는 추상적 개념
- `DBMS`는 그것을 실현하기 위해 작성된 구체적 소프트웨어

#### 소프트웨어와 데이터베이스

- 시스템은 단순히 데이터베이스만으로 구축할 수 없음
- 여러가지 소프트웨어와 조합하여 만드는데 이 작업은 SI (System Integration)
- 사용되는 소프트웨어는 크게 3가지 : 운영체제, 미들웨어, 애플리케이션
  - DBMS는 운영체제와 애플리케이션 중간인 **미들웨어**에 위치

#### 운영체제

- 시스템을 동작하게 하기 위한 일종의 토대가 되는 기능을 제공하는 소프트웨어
  - ex) Windows, OS X, iOS, Android
  - 자주 사용되는 OS : Windows(Windows Server), Linux(Red Hat, CentOS), UNIX(HP-UX, Solaris, AIX)

#### 미들웨어

- 데이터베이스는 여기에 속함. 데이터베이스는 OS에 설치하여 움직이는 것
- OS와 DBMS의 조합을 선택할 때 고려하는 것 : 예산, 제품기능, 개발자와 운용자의 기술조합

#### 제품에 따라 기능이 다름

- OS와 DBMS는 제품에 따라 기능이나 조작 방법이 다름
- **마이그레이션(Migration)** : 하나의 조합에서 다른 조합으로 시스템을 변경하는 것
  - ex) Oracle/Red Hat -> Oracle/Solaris (OS만 이행)
  - ex) Oracle/Windows -> SQL Server/Windows (DBMS만 이행)

#### 애플리케이션

- 업무용 기능을 가진 프로그램된 소프트웨어
- 사용자가 가장 빈번하게 조작하는 소프트웨어

#### 애플리케이션과 데이터베이스의 관계

- 사용자가 데이터베이스를 직접 조작하는게 아니라 **애플리케이션을 매개로 데이터베이스에 접근**함
  - 사용자 입장에선 데이터베이스에 접속하고 있다는 것도 의식하지 않는 경우가 대부분
  - 데이터베이스의 보안을 높이고, 업무로직을 애플리케이션에 집중시키기 위해
