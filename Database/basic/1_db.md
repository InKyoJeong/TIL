## 데이터베이스의 기본 기능

#### 데이터의 검색과 갱신

- 갱신: 등록, 수정, 제거
- 데이터 포맷에 유의 : 데이터베이스를 조작할때 데이터를 어떤 포맷(형식)으로 관리하는가
- 처리 성능에 유의 : 어느 정도 빠르기로 처리 가능한가

#### 동시성 제어

- 데이터베이스는 동시에 복수의 사용자로부터 검색이나 갱신 처리를 받음
  - 갱신의 무결성을 어느정도로 보장하는가

> Dirty Write : 어떤 사람도 문제없이 파일을 여는 것이 가능하고 나중에 수행된 쪽의 갱신이 반영되는 경우. (이런 식의 제어는 데이터의 무결성 관점에서 기피하는 경향이 있음)

#### 장애 대응

- 데이터 소실 문제 대책
  - 데이터 다중화 : 데이터를 한 곳이 아니라 복수의 장소에 분산 유지
  - 백업 : 데이터 소실이 발생했을때 데이터를 복원하는 방법

#### 보안

- 데이터베이스는 사용자로부터 가능한 보이지 않게 설계됨
  - 사용자는 서버를 의식할 필요가 없음
  - 데이터베이스는 기밀성이 높음

<br>

## 데이터베이스의 종류

#### 계층형 데이터베이스

- 데이터를 계층 구조로 관리하는 데이터베이스
  - ex) 조직도나 전체 구조도

#### 관계형 데이터베이스

- 2차원 표 형식으로 데이터를 관리. 가장 주류

#### 객체지향 데이터베이스와 XML 데이터베이스

- '객체'와 'XML'이라는 형식으로 데이터를 관리

#### NoSQL 데이터베이스

- SQL이란 관계형 데이터베이스를 만들기 위한 언어

<br>

## 관계형 데이터베이스란

<!-- - DBMS(DataBase Management System): 데이터베이스의 기능을 제공하는 소프트웨어 -->

- 관계형 데이터베이스의 '관계'는 2차원 표를 표기할때 사용하는 단어
  - 사람들이 자연스럽고 직관적으로 이해하기 쉽게 데이터를 관리할 수 있음
- 관계형 데이터베이스를 사용하려면 특정 소프트웨어 제품을 설치해야함
  - Oracle, MySQL, SQL Server 등이 있음

#### DBMS와 데이터베이스의 차이

- DBMS(DataBase Management System): 데이터베이스의 기능을 제공하는 소프트웨어
- `데이터베이스`는 기능이나 구조를 나타내는 추상적인 개념
  - `DBMS`는 그것을 실현하기위한 구체적인 소프트웨어
