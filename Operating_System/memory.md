## 메모리 관리

메모리는 실행될 프로그램을 위한 적재 장소로서 크기가 디스크보다 매우작다. 따라서 디스크에 있는 많은 프로그램중 몇개를 메모리 어디에 어떻게 적재할 것인가가 중요하다.

<br>

## 메모리의 구성

- 메모리 구성에 따라 적절한 관리 기법이 검토됨
- 구성과 관련하여 정해져야 할 것들
  - 다중 프로그래밍의 정도 : 한 번에 하나의 사용자 프로그램만 메모리에 있을 수 있게 할것인가 or 여러 프로그램이 같이 있도록 할 것인가
    - 후자가 다중 프로그래밍이며, 정도(Degree)란 메모리에 있는 프로세스 개수.
  - 프로세스에 메모리를 할당할 때 연속적으로 할 것인가 or 비연속적으로 할 것인가

### 메모리 구성에 따른 기법

- 적재 기법 : 프로세스에게 언제 메모리를 할당해 줄 것인가
- 배치 기법 : 프로세스들을 메모리공간의 어디에 적재할 것인가
- 교체 기법 : 메모리 공간이 부족할때 메모리에 있는 프로세스 중 어떤것을 디스크로 내보내고 공간을 확보할 것인가
- 할당 기법 : 프로세스에게 메모리 공간을 얼마 정도로 줄 것인가

<br>

## 단일 프로그래밍

- 한 번에 하나의 프로세스만이 메모리에 적재되고, 실행이 종료되면 다음 프로세스가 적재되는 시스템

#### 문제점

- 메모리 낭비와 CPU와 다른 자원들의 낭비로 시스템 성능이 떨어짐

<br>

## 고정 분할에서의 다중 프로그래밍

- 메모리를 여러개의 분할로 나눠놓고, 각 분할에 하나의 프로세스만을 수용하게 해서 다중프로그래밍을 구현하는 방식

#### 문제점

- 가장 큰 분할보다 더 큰 프로그램의 수용 문제
- 단편화(Fragmentation)
  - 내부 단편화 : 분할내에서 프로세스를 수용하고 남는 낭비공간
  - 외부 단편화 : 분할 크기가 작아서 프로세스들을 수용못할때 통째로 분할이 낭비되는것

<br>

## 가변 분할에서의 다중 프로그래밍

- 가변 분할 : 분할의 시기/개수/크기에 상관없이 프로세스를 수용할 때 그 크기만큼 메모리 공간을 할당
- 가변 분할 방식으로 메모리를 관리하면 홀(Hole)이 생겨 빈 공간을 병합하는 작업이 요구됨
  - 인접한 빈 공간 병합 : 빈 공간으로 반납될 때 인접한 빈공간을 합침
  - 빈 공간 전부의 통합 : 사용중인 공간들을 메모리 한쪽으로 옮기고 빈공간을 모두합침 -> 모든 프로세스 재배치로 상당한 시간, 통합 진행되는 동안 프로세스들의 실행중지 발생

> 홀(Hole) : 빈 공간이긴 하지만 크기가 아주작아서 할당될 가능성이 적기때문에 낭비되는 공간

<br>

## 버디 메모리 관리
